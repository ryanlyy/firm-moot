# AI 法庭模拟器

利用大语言模型模拟即将到来的庭审，帮助律师进行充分的庭前对抗演练，发现论证盲区，优化辩论策略。

## 核心功能

### 1. 案件管理
录入完整的案件信息（事实、诉求、证据、法律依据等），支持上传庭审笔录（Word / TXT），作为 AI 模拟的上下文基础。

### 2. 庭审记录
在案件下管理真实庭审记录，支持 `.docx` Word 文件自动提取文字、`.txt` 直接导入或手动输入。记录会被自动注入后续模拟和分析的 AI 上下文中。

### 3. 庭审模拟（四种模式）

| 模式 | 说明 |
|------|------|
| **对抗辩论** | AI 扮演对方律师，进行激烈的法庭辩论对抗 |
| **完整庭审** | AI 同时扮演审判长 + 对方律师，模拟完整庭审流程 |
| **证人质询** | AI 扮演案件相关证人，练习交叉质询技巧 |
| **即时分析** | 输入论点，AI 从对方视角实时点评、反馈 |

### 4. 策略分析

| 分析类型 | 说明 |
|----------|------|
| **全面分析** | 争议焦点、优劣势、证据链、法律适用的全方位评估 |
| **风险识别** | 识别我方论证的薄弱环节和对方可能的攻击方向 |
| **庭审策略** | 从开庭陈述到结辩的全流程策略建议 |
| **问题预测** | 预测法官可能提出的关键问题及回答策略 |

### 5. 历史对比与持续改进
- 自动保存每次模拟会话和策略分析结果
- 再次分析时自动对比历史记录，给出增量改进建议
- 庭审记录 + 历史分析共同作为数据源，逐步提升模拟的准确性和完整性

### 6. 用户与权限管理
- 管理员 / 普通用户角色区分
- 普通用户仅能查看自己的案件数据，管理员可查看全部并拥有统计功能
- 5 分钟无操作自动退出登录

### 7. 系统管理（管理员）
- 数据监控：数据库大小、连接池、运行时长
- 数据统计：用户/案件/模拟的汇总与分布
- 备份恢复：手动备份、定时备份、一键恢复

## 快速开始

### 前提条件
- Python 3.10+
- PostgreSQL 16+（可通过 `scripts/setup.bat` 自动安装）

### 方式一：一键安装（Windows）

双击运行 `scripts/setup.bat`，脚本会自动完成：
1. 检测并安装 Python 依赖
2. 下载安装 PostgreSQL（如未安装）
3. 创建数据库用户和数据库
4. 生成 `.env` 配置文件
5. 启动应用服务

### 方式二：手动安装

```bash
# 1. 安装依赖
pip install -r requirements.txt

# 2. 配置环境变量
cp .env.example .env
# 编辑 .env，填入数据库连接和 LLM 配置

# 3. 启动服务
python run.py
```

访问 http://127.0.0.1:8000 ，默认管理员账号 `admin` / `admin123`。

### 方式三：Docker 部署

```bash
# 复制并编辑环境变量
cp .env.example .env

# 启动所有服务（PostgreSQL + App + Nginx）
docker compose up -d
```

## 使用建议

1. **案件信息越详细，模拟效果越好** —— 尽量填写完整的事实、证据和法律依据
2. **上传庭审记录** —— 真实笔录能显著提升 AI 模拟的针对性
3. **先做策略分析，再进入模拟** —— 了解自身优劣势后再进行对抗练习
4. **多轮对抗** —— 在模拟中反复推敲论点，每一轮都尝试不同的辩论角度
5. **关注法官问题预测** —— 提前准备法官可能提出的问题，避免庭审被动
6. **反复分析** —— 每次分析都会参考历史记录，给出递进式改进建议

## 技术栈

- **后端**：Python / FastAPI / Uvicorn
- **数据库**：PostgreSQL（asyncpg 异步驱动）
- **认证**：JWT（python-jose + bcrypt）
- **AI**：OpenAI 兼容 API（Ollama / GPT / DeepSeek 等）
- **前端**：原生 HTML + Tailwind CSS + JavaScript
- **部署**：Docker Compose + Nginx 反向代理
- **备份**：pg_dump / psql + APScheduler 定时任务
